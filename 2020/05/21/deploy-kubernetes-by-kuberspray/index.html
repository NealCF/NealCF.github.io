Deploy kubernetes by kuberspray

前言

​ 部署Kubernetes除了手动方式外，还有诸如Kubeadm、Kubespray、Breeze、Rancher、kargo等多种自动化方式。工具没有好坏之分，能干事、效率高就行。这里，笔者仍使用Kubespray部署当前K8s最新版本（用着真的很贴身），可自动化部署HA集群、可灵活定制开发、高稳定性等。

环境说明

IP 主机名 角色
192.168.10.100 Ansible-client ansible的部署节点
192.168.10.101 Kube-master1 Master,node
192.168.10.102 Kube-master2 Master,node
192.168.10.103 Kube-master3 Master,node

Kubernetes版本

1.15.0

环境准备

  1. 关闭selinux(all)
    sed -i '/^SELINUX=/s/permissive/disabled/' /etc/selinux/config

  2. 关闭fiewalld(all)
    systemctl stop firewalld && systemctl disable firewalld

  3. 升级内核(k8s)

    1. 载入公钥
      rpm –import https://www.elrepo.org/RPM-GPG-KEY-elrepo.org

    2. 升级安装ELRepo
      rpm -Uvh http://www.elrepo.org/elrepo-release-7.0-3.el7.elrepo.noarch.rpm

    3. 载入elrepo-kernel元数据
      yum –disablerepo=* –enablerepo=elrepo-kernel repolist

    4. 查看可用的rpm包
      yum –disablerepo=* –enablerepo=elrepo-kernel list kernel*

      说明：

      lt  ：long term support，长期支持版本；
ml：mainline，主线版本；

    5. 安装最新版本的kernel
      yum –disablerepo=* –enablerepo=elrepo-kernel install kernel-ml.>x86_64 -y

    6. 删除旧版本工具包
      yum remove kernel-tools-libs.x86_64 kernel-tools.x86_64 -y

    7. 安装新版本工具包
      yum –disablerepo=* –enablerepo=elrepo-kernel install kernel-ml-tools.x86_64 -y

    8. 查看内核插入顺序
      [root@server-1 ~]# awk -F ' ‘$1==”menuentry “ {print i++ “ : “ $2}’ /etc/grub2.cfg
      0 : CentOS Linux (4.20.12-1.el7.elrepo.x86_64) 7 (Core)
      1 : CentOS Linux (3.10.0-957.5.1.el7.x86_64) 7 (Core)
      2 : CentOS Linux (3.10.0-957.el7.x86_64) 7 (Core)
      3 : CentOS Linux (0-rescue-ca0f6fb3c5f24478abc0a2e275281d7a) 7 (Core)

    9. 查看当前实际启动顺序
      grub2-editenv list

    10. 设置默认启动
      grub2-set-default 0
      grub2-editenv list

    11. reboot

    12. 卸载掉旧的kernel

  4. 安装ansible(ansible-client)

yum install -y epel-release
yum install -y python-pip ansible python36 python36-pip
pip install --upgrade jinja2
pip3 install --upgrade jinja2

配置kubespray(ansible-client)

线上不使用master分支，建议切换至最新tag
地址： https://kubespray.io

  1. 安装git

    yum install git -y
git clone https://github.com/kubernetes-incubator/kubespray.git
cd kubespray/
git branch -va 查看remote的分支
git checkout -b v版本号 remote/../.. 选一个新的，bug可能少

  2. 安装requirements.txt中的依赖

    pip install -r requirements.txt

  3. 复制inventory模板作为自己的集群inventory文件

    cp -r inventory/sample inventory/mycluster

  4. 使用inventory_builder，初始化inventory文件

    IPS=(192.168.10.101 192.168.10.102 192.168.10.103)
CONFIG_FILE=inventory/mycluster/hosts.yml
python3 contrib/inventory_builder/inventory.py ${IPS[@]}

    设置两个环境变量IPS和CONFIG_FILE 代入inventory.py的脚本来生成hosts.yml文件

  5. 确认inventory文件生成无误

    cat inventory/mycluster/hosts.yml 

[all]
k8s-master1    ansible_host=192.168.10.101 ip=192.168.10.101
k8s-master2    ansible_host=192.168.10.102 ip=192.168.10.102
k8s-master3    ansible_host=192.168.10.103 ip=192.168.10.103

[kube-master]
k8s-master1
k8s-master2
k8s-master3

[kube-node]
k8s-master1
k8s-master2
k8s-master3

[etcd]
k8s-master1
k8s-master2
k8s-master3

[k8s-cluster:children]
kube-node
kube-master

[calico-rr]

    Note:

    稍加修改主机名，如我的kube-master1-3，安装过程中会自动修改主机名和node master名称。
    生产环境，master和etcd 请配置在不同节点。

    这里是测试环境，因硬件资源紧张就将master和etcd部署在相同节点了。

  6. 查看或修改默认参数

    at inventory/mycluster/group_vars/k8s-cluster/k8s-cluster.yml

kube_version: v1.15.0   / 我修改为1.15.0，镜像缓存网站可能没有最新的镜像
kube_network_plugin: calico
kube_service_addresses: 10.233.0.0/18
ube_pods_subnet: 10.233.64.0/18
kube_proxy_mode: ipvs

修改代理(kubespray)

代理网站地址： Azure中国Github Azure中国Web

把脚本中的docker.io、grc.io和quay.io地址修改成azure中国的地址：

sed -i 's@gcr\.io@gcr\.azk8s\.cn@g'  role/download/defaults
sed -i 's@docker\.io@dockerhub\.azk8s\.cn@g' role/download/defaults
sed -i 's@quay\.io@quay\.azk8s\.cn@g' role/download/defaults
sed -i 's@gcr\.io@gcr\.azk8s\.cn@g' inventory/mycluster/group_vars/k8s-cluster/k8s-cluster.yml

配置内核参数(k8s)

添加内核参数开启路由转发等功能
cat << eof
net.ipv4.ip_forward=1
net.ipv4.ip_local_reserved_ports=30000-32767
net.bridge.bridge-nf-call-iptables=1
net.bridge.bridge-nf-call-arptables=1
net.bridge.bridge-nf-call-ip6tables=1
eof >> /etc/sysctl.conf
sysctl -p

部署集群(kubespray)

ansible-playbook -i inventory/mycluster/hosts.yml cluster.yml -b -v

验证(k8s)

[root@kube-master1 ~]# kubectl get node
NAME           STATUS   ROLES    AGE   VERSION
kube-master1   Ready    master   72m   v1.15.0
kube-master2   Ready    master   73m   v1.15.0
kube-master3   Ready    master   72m   v1.15.0

Note:

可以看到节点都是ready状态了，说明部署成功。

部署测试

# 创建一个nginx
kubectl create nginx --image=nginx:1.7.9 --port=80 

# 暴露nginx端口
kubectl expose deployment nginx --type=NodePort

# 查看nginx服务详情 
kubectl get svc nginx

nginx   NodePort   10.233.22.159   <none>        80:31758/TCP   5m58s

# 访问测试
curl 192.168.10.101:31758

高可用

etcd高可用

etcd本身就支持集群模式，所以啥都不用考虑，只要保证节点数量足够，升级备份之类的事情，kubespray已经帮你做了。

由于etcd采用Raft一致性算法，集群规模最好不要超过9个，推荐3，5，7，9个数量。具体看集群规模。如果性能不够，宁可多分配资源，也最好不要超过9个。

这里能选择偶数个节点吗？ 最好不要这样。原因有二：

  • 偶数个节点集群不可用风险更高，表现在选主过程中，有较大概率或等额选票，从而触发下一轮选举。
  • 偶数个节点集群在某些网络分割的场景下无法正常工作。试想，当网络分割发生后，将集群节点对半分割开。此时集群将无法工作。按照RAFT协议，此时集群写操作无法使得大多数节点同意，从而导致写失败，集群无法正常工作。

kube-apiserver 高可用
https://kubespray.io/#/docs/ha-mode
haproxy

yum install -y haproxy keepalived
vim /etc/haproxy/haproxy.cfg 
listen kubernetes-apiserver-https
  bind *:8443
  option ssl-hello-chk
  mode tcp
  timeout client 3h
  timeout server 3h
  server master1 192.168.10.81:6443
  server master2 192.168.10.82:6443
  server master3 192.168.10.83:6443
  balance roundrobin

维护命令

如果需要扩容Work节点，则修改hosts.yml文件，增加新增的机器信息 执行命令：
ansible-playbook -i inventory/mycluster/hosts.yml scale.yml -b -v -k

将hosts.yml文件中的master和etcd的机器增加到多台，执行部署命令
ansible-playbook -i inventory/mycluster/hosts.yml cluster.yml -b -vvv

刪除节点，如果不指定节点就是刪除整个集群：
ansible-playbook -i inventory/mycluster/hosts.yml remove-node.yml -b -v

卸载执行命令：
ansible-playbook -i inventory/mycluster/hosts.yml reset.yml -b -vvv

升级K8s集群，选择对应的k8s版本信息，执行升级命令。涉及文件为upgrade-cluster.yml。
ansible-playbook upgrade-cluster.yml -b -i inventory/mycluster/hosts.yml -e kube_version=vX.XX.XX -vvv
